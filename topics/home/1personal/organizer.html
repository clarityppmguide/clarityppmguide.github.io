Organizer

Organizer Page provides user to have access to "Actions Items", "Tasks", "Processes" and "Notifications".

Below are the 4 tabs that Organizer page holds.

1. Action Items -

Project action items are reminders about work that you assign to yourself or to others and that others assign to you.

  • You can use action items to track the progress of projects and to help ensure that a project is complete and on time.
  • Create project-related action items from within a project.
  • When you create an action item, you become the owner of the action item or you can assign it to resources that access the item.
  • As the owner, you can modify and delete it.

2. Task -

This Task tab will display the list of Tasks that logged in user is assigned to.



3. Processes -

This tab will have two views "Available", "Initiated", both will list processes, by Default we land into "Initiated" tab

- Initiated :-

  • This section will show the list of processes that were Initiated in the background and their status, start time and finish time, Initiated by.
  • We could also check the Status of the Process, Cause of Error and Information about process failure.

- Available :-

  • This section will show the list of processes that are Available in the Application.
  • One could trigger the processes from the same page by selecting the Process and click on Start.

4. Notifications -

The occurrence of certain events in other parts of the software can trigger automatic notifications, The Notifications are shown in this tab.

Around 15 different Functional Areas related Notifications are shown here. ( like Timesheet, Action Items, Change requests, Issues and etc,.)

Navigation

Please follow the below steps

  • Navigate to Home tab
  • Under Personal click on Organizer link

Click the images to see the view

Query

For Developers

Following Query is used under the Task Tab 

SELECT
       PR.PRNAME TASK,
       INV.NAME INVESTMENT,
       PR.PRSTART START,
       PR.PRFINISH FINISH,
       (CASE ASG.PRSTATUS WHEN 0 THEN 'Not Started' WHEN 1 THEN 'Started' WHEN 2 THEN 'Not-Started' END) ASSIGNMENTSTATUS


      FROM INV_INVESTMENTS INV
      JOIN PRTASK PR ON INV.ID = PR.PRPROJECTID
      JOIN PRASSIGNMENT ASG ON ASG.PRTASKID = PR.PRID
      JOIN SRM_RESOURCES SRM ON SRM.ID=ASG.PRRESOURCEID
WHERE
      SRM.UNIQUE_NAME = 'admin'
      AND INV.IS_ACTIVE = 1

Query

For Developers

Following Query is used under the Process Tab --> Initiated 


                          SELECT
                          V.NAME PROCESS
                          ,V.PROCESS_CODE ID
                          ,OBJ.OBJECT_TYPE_CODE PRIMARYOBJECT
                          ,OBJ.OBJECT_NAME OBJECTNAME
                          ,RUN.PERCENT_COMPLETE PROGRESS
                          ,STEPV.NAME STEPSINPROGRESS
                          ,(SELECT V.NAME FROM CMN_LOOKUPS_V V WHERE V.LANGUAGE_CODE='en' AND V.LOOKUP_TYPE='BPM_PROCESS_INSTANCE_STATES' AND V.LOOKUP_CODE=RUN.STATUS_CODE) PROCESSSTATUS
                          ,(SELECT FULL_NAME FROM SRM_RESOURCES SRM WHERE SRM.USER_ID=RUN.INITIATED_BY) INITIATEDBY
                          ,RUN.START_DATE STARTDATE
                          ,RUN.END_DATE FINISHDATE
                          FROM
                                BPM_DEF_PROCESSES_V V
                          JOIN  BPM_DEF_PROCESS_VERSIONS VER ON VER.PROCESS_ID= V.ID
                          JOIN  BPM_RUN_PROCESSES RUN ON RUN.PROCESS_VERSION_ID = VER.ID
                          JOIN BPM_DEF_OBJECTS OBJ ON OBJ.PK_ID=VER.ID
                          JOIN BPM_RUN_STEPS RUNS ON RUNS.PROCESS_INSTANCE_ID=RUN.ID
                          JOIN BPM_DEF_STEPS_V STEPV ON STEPV.ID=RUNS.STEP_ID
                          WHERE
                          V.LANGUAGE_CODE='en'
                          AND OBJ.TABLE_NAME='BPM_DEF_PROCESS_VERSIONS'
                          AND OBJ.OBJECT_TYPE_CODE NOT IN ('step','process')
                          AND STEPV.LANGUAGE_CODE='en'
