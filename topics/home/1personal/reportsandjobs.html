Reports and Jobs

The CA PPM allows users to Run or Schedule the Report and Jobs that produce some results which favors their daily business.

Please check the below provided information to know more about Reports and Jobs

Navigation

Please follow the below steps

  • Navigate to Home tab
  • Under Personal click on Report and Jobs link

Click the images to see the view

Reports -

In CA PPM a Report is a document that which provides information in organized format for various or specific audience.

The various formats are like CSV, XLSX, Excel, PPT, PDF and etc,.

Access Rights

For Developers

Access Rights are considered as permission to view or access particular section in CA PPM

Reports - Access Allows user access to the reports pages. This right is dependent on either the Reports - Run - All right or the user being granted instance level rights such as Report - Run, Report - View Output or Report - Edit Properties.
Reports - Run - All Allows user to run any report. Allows scheduling of any report, editing of the report properties for any report and viewing the output of any report. This right is dependent on Reports - Access being granted.

Tables

For Developers

Below are the tables used for this View - Click the Image to view.

1.CMN_SCH_JOB_DEFINITIONS This Table stores Jobs and Reports data like it's Name, ID, type, Active and etc,..
2.CMN_LOOKUPS_V This table stores the the values for a static lookup type.

Query

For Developers

Following Query is used to fetch the data from View 

                    
                    SELECT
                    		CMN.NAME
                    		,CMN.DESCRIPTION
                    FROM
                    		CMN_SCH_JOB_DEFINITIONS DEF
                    		JOIN CMN_LOOKUPS_V CLV ON CLV.ID=DEF.JOB_TYPE AND CLV.LOOKUP_TYPE='SCH_JOB_TYPE' AND CLV.LANGUAGE_CODE='en'
                    		JOIN CMN_CAPTIONS_NLS CMN ON CMN.PK_ID=DEF.ID
                    WHERE 	CLV.LOOKUP_CODE='REPORT' AND CMN.LANGUAGE_CODE='en'
                    		AND CMN.TABLE_NAME = 'CMN_SCH_JOB_DEFINITIONS'

Jobs -

Jobs are sets of actions that you can run or schedule to automate common administrative processes.

For example, jobs can populate or refresh database tables and run background processes.A job can include queries, statements, or procedures.

Access Rights

For Developers

Access Rights are considered as permission to view or access particular section in CA PPM

Jobs - Access Allows user access to the jobs pages. This right is dependent on either the Jobs - Run - All right or the user being granted instance level rights such as Job - Run, Job - View Output or Job - Edit Properties.
Jobs - Run - All Allows user to run any job. Allows scheduling of any job, editing of the job properties for any job and viewing the output of any job. This right is dependent on Jobs - Access being granted.

Tables

For Developers

Below are the tables used for this View - Click the Image to view.

1.CMN_SCH_JOB_DEFINITIONS This Table stores Jobs and Reports data like it's Name, ID, type, Active and etc,..
2.CMN_LOOKUPS_V This table stores the the values for a static lookup type.

Query

For Developers

Following Query is used to fetch the data from View 

                    
SELECT
CMN.NAME JOBDEFINITIONS
,BDP.JOB_CODE ID
,CMN.DESCRIPTION DESCRIPTION
,DECODE(BDP.JOB_TYPE,50317,'Java'
,50318,'XBL'
,50319,'PMD'
,50375,'SQL Stored Procedure'
,5009912,'ETL'
,5012355,'Report'
) JOBTYPE
,BDP.EXECUTABLE FILENAME
,DECODE(BDP.IS_SYSTEM,0,'User-defined',1,'System',2,'System-restricted') SYSTEMTYPE
,BDP.IS_ACTIVE ACTIVE

FROM CMN_SCH_JOB_DEFINITIONS BDP
JOIN CMN_CAPTIONS_NLS CMN ON CMN.PK_ID=BDP.ID WHERE CMN.LANGUAGE_CODE='EN'
AND CMN.TABLE_NAME = 'CMN_SCH_JOB_DEFINITIONS'
ORDER BY CMN.NAME
